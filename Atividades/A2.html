Processo de Testes

Os processos de testam a funcionalidade, confiabilidade e performace dos softwares. Estes são requisitos básicos. Os testes possuem uma organização para acontecer, ou seja, é comum que haja um padrão nas realizações desses processos:

  • Procedimentos iniciais:

    nessa etapa é estabelicido um acordo entre as partes envolvidas.No acordo são esbelecidas as regras para se fazer os testes;

  • Planejamento:

    nesta etapa acontece o desenvolvimento e a revisão do plano de estratégia de testes;

  • Preparação:

    é a formação da equipe que irá realizar os testes e os equipamentos que serão utilizados;

  • Especificação:

    são construídos os Casos de Testes, os códigos para testes automatizados são escritos e testes que revisam a documentação de especificações do software;

  • Execução:

    começa a fase de testes, onde se segue o roteiro pré-definido na fase de especifícações e os resultados são documentados;

  • Entrega:

    fase em que todos os testes foram realizados e o software já pode ser disponibilizado no ambiente de produção.

Como a manutenção de um software requer altos custos, a fase de testes fica junto com a fase de desenvolvimento.
