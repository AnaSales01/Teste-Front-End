Frameworks

  1. React
    • Vantagens:

      Flexível, é livre para definir padão, ou seja, não existe uma estrutura obrigatória para se seguir;
      Browser Tools, ferramenta que auxiliam no debug e construção das aplicações direto no browser;
      Fácil de migrar entre versões;
      Mobile com react native.

    • Desvantagens:

      Documentação oficial reduzida;
      HTML, CSS e JavaScript no mesmo documento;
      É livre para definir padão, o que pode gerar projetos poucos escaláveis, caso não seja estudado e se procure uma boa estrutura de projeto.

    • Sistemas que utilizam:

      É utilizado pela Facebook, Netflix, Airbnb, Walmart, SeatGeek, Uber, Spotify, Americanas, entre outros.

  2. Angular
    • Vantagens:

      É um framework robusto, ou seja, quase tudo que é preciso para uma aplicação se encontra nele nativamente.
      Mais tempo de mercado, pois foi criado em 2009.
      Documentação detalhada.

    • Desvantagens:

      Dificuldade de migração entre versões;
      TypeScript, esse ponto pode ser visto como um ponto negativo, já que ele é um pouco complexo para quem está começando;
      Perda de popularidade.

    • Sistemas que utilizam:

      É utilizado pelo Azure, Santander, Toro, Forbes e BMW.

  3. Vue
    • Vantagens:

      É um framework progressivo extremamente leve e completo;
      Documentação completa que tem conceitos, exemplos e dicas;
      Fácil aprendizagem;
      Adaptável (fácil migração para React e Angular).

    • Desvantagens:

      Como é um framework relativamente novo tem menos recursos prontos na comunidade;
      Diretivas, como o angular podem ser um pouco complicadas para se aprender e utilizar no template HTML;
      Muito flexível, como o react é livre para seguir padrões de projeto.

    • Sistemas que utilizam:

      É utilizado pela Glovo, GitLab, Renner, Utorrent, entre outros
