Pesquisa sobre testes e metodologia XP

TDD: fundamentos do desenvolvimento orientado a testes

Os testes nos sistemas têm que ser pensados antes do processo de desenvolvimento, pois ao passar muito tempo o custo é bem maior, ou seja, se deixar para fazer os testes depois que o sistema já foi desenvolvido ou depois que foi lançado no mercado os custos podem ser 10 vezes maior.

TDD (Test Driven Development/ Desenvolvimento orientado a teste) é parte da metodologia XP. Para desenvolver esse teste é preciso seguir 3 fazes:

  1. Escrever um teste que falhe
  2. Escrever um código para passar no teste
  3. Eliminar a redundância

Integrando XP as principais metodologias ágeis

O XP ( eXtreme Programming) é uma metologia para times pequenos a médio, que desenvolvem aplicações que se modificam rapidamente.

O processo inicia por um plano geral, ou seja, entender o problema como um todo, mas sem muitos detalhes e, posteriormente, reparte o problema em partes, para dar início ao seu desenvolvimento.
